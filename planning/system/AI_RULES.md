# AI RULES

This document defines rules that any AI agent must follow when interacting with this planning repository.

---

# General Rules

1. Always read **SYSTEM_MASTER_INDEX.md** before making any modification.

2. Never modify the architecture of the system without documenting the change in DECISION_LOG.md.

3. When creating or modifying modules, always update the related documentation.

4. Maintain consistency between system modules, UI specifications, and database planning.

5. Never delete historical documentation unless explicitly instructed.

6. When proposing major changes, create a documented proposal before applying the modification.

---

# Planning Integrity

AI agents must ensure that:

- system modules remain logically separated
- module dependencies remain valid
- documentation remains synchronized with system design

---

# Documentation Principle

The planning repository is the **source of truth** for system design.

All system decisions must be documented before implementation.
